General Notes

Important Terms

html = HyperText Markup Language, the primary coding language and main type of document of the Internet

http = Hypertext Transfer Protocol, the method by which a browser (at the direction of a computer) requests html files from a server

elements are the basic building blocks of html code; most are further customized with tags and attributes

Basic Concepts

The Web is a vast network of tubes. Actually, links. Pages linked to each other form the Internet. Basic pages formed from html documents and text contain more complex files such as images, audio and video.

html is a language for building webpages out of boxes, or blocks. The blocks are usually made of smaller blocks, and painted in different ways, and they may even have contents of their own, but the most basic piece of a page will always be a block. Pages are built out of painted blocks and then connected by links. When turning a page concept into html a designer will often break the concept down into the most basic blocks, or "boxify" it, to visualize the underlying structure and then attack it in pieces.

Computers are dumb. While normal conversation allows for a fair degree of ambiguity, relying on the listener to use context clues and their own knowledge to understand your meaning, a computer will interpret your language as literally as possible. This is the source of many issues between programmers and computers.

In html, whitespace such as extra spaces and new lines will be condensed when the code is actually rendered. This is the reason line breaks are used.

Tag Basics

First rule of tag club: An unclosed tag will mark up everything after the opening tag.

Second rule of tag club: IT IS NOT IMPORTANT TO REMEMBER THE ATTRIBUTES OF EVERY TAG. There are too many tags and attributes for remembering all of them to be practical. When developing in html, it's appropriate to use online resources such as W3Schools or the Mozilla Developer Network.

b is a tag for bolding text; em is a tag to emphasize text with italics.

a is an anchor tag, for making links.

href is an attribute of the anchor tag; the value should be the URL of the link.

img is the tag; src is an attribute for the source of the image; alt is an attribute for alt text, which will load if the image does not.

(img is a void tag, or a tag that has no content and therefore needs no closing tag)

br is a break tag, to break up lines of text.

Text can also be broken up with p, the paragraph tag; p wraps, so it needs content and a closing tag.

While br is simply ending a line, p is creating a whole new element which can have its own attributes like height (a block tag).

span and div create container elements, with div creating a box and span creating an inline element.These tags are useful for assigning a css class to text, making it easier to add character to your page.

HTML Documents

Doctypes in html used to be hellish; html5 has simplified it a great deal. Now the doctype html is sufficient for any html document.

In an actual html document, the header will contain things like metadata, css and the title of the page.

The actual content of the document will be in the body section, sometimes broken up under different headings.

Code Structure

A new indent signals the beginning of a new idea, but a new idea will increase the indentation level, decreasing it when the idea is complete.

The doctype declares what language the browser should parse the document in, in this case html.

The html tag opens the actual code of the document, under which there will be a header, a body and a closing html tag.
